00052 /*
00053  * New world logging!
00054  * The brief summary:
00055  *  Logging used to be a simple affair, a FILE * handled by a nonblocking logging class inheriting from EventHandler, that was inserted
00056  *  into the socket engine, and wrote lines. If nofork was on, it was printf()'d.
00057  *
00058  *  We decided to horribly overcomplicate matters, and create vastly customisable logging. LogManager and LogStream form the visible basis
00059  *  of the new interface. Basically, a LogStream can be inherited to do different things with logging output. We inherit from it once in core
00060  *  to create a FileLogStream, that writes to a file, for example. Different LogStreams can hook different types of log messages, and different
00061  *  levels of output too, for extreme customisation. Multiple LogStreams can hook the same message/levels of output, meaning that e.g. output
00062  *  can go to a channel as well as a file.
00063  *
00064  *  HOW THIS WORKS
00065  *   LogManager handles all instances of LogStreams, classes derived from LogStream are instantiated and passed to it.
00066  */
00067
