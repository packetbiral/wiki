00024 /*
00025  * The server list in InspIRCd is maintained as two structures
00026  * which hold the data in different ways. Most of the time, we
00027  * want to very quicky obtain three pieces of information:
00028  *
00029  * (1) The information on a server
00030  * (2) The information on the server we must send data through
00031  *     to actually REACH the server we're after
00032  * (3) Potentially, the child/parent objects of this server
00033  *
00034  * The InspIRCd spanning protocol provides easy access to these
00035  * by storing the data firstly in a recursive structure, where
00036  * each item references its parent item, and a dynamic list
00037  * of child items, and another structure which stores the items
00038  * hashed, linearly. This means that if we want to find a server
00039  * by name quickly, we can look it up in the hash, avoiding
00040  * any O(n) lookups. If however, during a split or sync, we want
00041  * to apply an operation to a server, and any of its child objects
00042  * we can resort to recursion to walk the tree structure.
00043  * Any socket can have one of five states at any one time.
00044  *
00045  * CONNECTING:  indicates an outbound socket which is
00046  *                                                      waiting to be writeable.
00047  * WAIT_AUTH_1: indicates the socket is outbound and
00048  *                                                      has successfully connected, but has not
00049  *                                                      yet sent and received SERVER strings.
00050  * WAIT_AUTH_2: indicates that the socket is inbound
00051  *                                                      but has not yet sent and received
00052  *                                                      SERVER strings.
00053  * CONNECTED:   represents a fully authorized, fully
00054  *                                                      connected server.
00055  * DYING:       represents a server that has had an error.
00056  */
00057 enum ServerState { CONNECTING, WAIT_AUTH_1, WAIT_AUTH_2, CONNECTED, DYING };
